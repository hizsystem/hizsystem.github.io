DAY 0 15분 · 설치 + 첫 인사

왜 Claude Code인가

웹 Claude는 "대화하는 챗봇"이지만, Claude Code는 내 컴퓨터를 맡기는 비서예요. 차이를 먼저 느끼는 게 1일차 동기부여의 90%입니다.

Why · 왜 배우나

웹 Claude vs Claude Code, 뭐가 다른가

항목웹/앱 ClaudeClaude Code (CLI)
대화 방식채팅창내 컴퓨터 폴더 안
파일업로드해서 보여줌직접 만들고 수정함
결과물대화창 안내 폴더에 쌓임
외부 연결제한적Slack, Notion, Figma 등 연결
반복 작업매번 새로"커맨드"로 1초 호출

한 줄 요약: 웹은 1회용 답변, Code는 누적되는 시스템이에요. 비개발자도 1년 쌓으면 회사 한 부서가 됩니다.

How · 어떻게 깔까

설치 (10분)

OS를 골라주세요. 결과물은 똑같아요.

1. 터미널을 엽니다

맥: ⌘ + Space → "터미널" 검색 → Enter. 까만 창이 뜨면 OK.

2. Node.js가 깔려있는지 확인

$ node --version

v20.x.x 같은 게 뜨면 OK. command not found가 뜨면 아래 한 줄로 깝니다.

# Homebrew 없으면 먼저 설치 $ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" $ brew install node

3. Claude Code 설치

$ npm install -g @anthropic-ai/claude-code

4. 동작 확인

$ claude --version

1. PowerShell을 엽니다

시작 메뉴 → "PowerShell" 검색 → 우클릭 → 관리자 권한으로 실행. 파란 창이 뜨면 OK.

2. Node.js LTS 설치

가장 쉬운 방법: nodejs.org에서 LTS Installer (.msi)를 받아 더블클릭. "다음 → 다음 → 완료"만 누르면 됩니다.

또는 PowerShell 한 줄로:

PS> winget install OpenJS.NodeJS.LTS

3. PowerShell을 새로 엽니다

중요: 기존 창은 새로 깐 Node를 못 봅니다. 꼭 닫고 새로 열어주세요.

PS> node --version

4. Claude Code 설치

PS> npm install -g @anthropic-ai/claude-code

5. 동작 확인

PS> claude --version
⚠️ 윈도우 함정 3가지
  • 한글 폴더명 (바탕화면, 내 문서)은 피하세요. 영문 폴더 권장.
  • 스크립트 실행 차단 에러가 나면 PowerShell에 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned 한 번 입력 → Y.
  • WSL은 추천하지 않아요. 비개발자에겐 너무 무겁고, PowerShell만으로 충분합니다.
🍎🪟 더 쉬운 길 (양쪽 공통)
터미널이 무서우면 claude.ai/code에서 데스크톱 앱을 받으세요. 똑같이 동작하고 GUI라 친숙해요. 단, 이 가이드는 CLI 기준입니다 (CLI가 결국 더 강해서요).

Stuck · 막히면
  • command not found: claude → npm 글로벌 경로 문제. npm prefix -g 결과를 PATH에 추가하거나, 데스크톱 앱으로 우회하세요.
  • 권한 거부 (EACCES) → sudo npm install ... 하지 마세요. 대신 공식 가이드대로 npm prefix를 바꾸는 게 안전합니다.
  • 로그인 단계 → 첫 실행 시 브라우저로 Anthropic 로그인 창이 뜹니다. 이메일로 로그인 → 토큰 자동 저장.
+ 알아두기 · API와 비용

"이거 돈 나가나요?" 솔직한 답

두 가지 결제 방식이 있고, 비개발자는 처음엔 (1)번부터 추천합니다.

(1) Claude.ai 구독(2) API 키
요금월 $20 (Pro) 정액쓴 만큼 (토큰당)
한도몇 시간마다 리셋되는 사용량무제한 (단 결제 발생)
장점안심, 예측 가능대량/자동화에 유리
단점한도 도달 시 잠시 막힘잘못 돌리면 폭탄 청구서
추천입문 ~ 일상 업무본격 자동화 (스킬 10개+)

토큰이 뭐예요?

AI가 글을 처리하는 단위예요. 한국어 1글자 ≈ 1.5토큰. 즉 한국어 1만자 ≈ 1.5만 토큰 정도라고 생각하면 편합니다.

모델별 가격 감 (Sonnet 기준)

  • 입력 100만 토큰 ≈ $3 (한국어 약 67만자)
  • 출력 100만 토큰 ≈ $15
  • 실사용: 하루 빡세게 굴려도 평균 $2~5. 한 달 풀로 써도 $50~100 수준.
💡 첫 달 추천 시나리오
Claude.ai Pro 구독 ($20)으로 시작 → 한 달 써보고 한도 자주 막히면 그때 API로 갈아타기. API로 가면 결제 한도(spending limit)를 꼭 설정하세요. console.anthropic.com에서 월 $20 같은 식으로 상한 캡 가능.
🚨 API 키 보안
API 키는 은행 계좌번호+비밀번호라고 생각하세요. GitHub, 슬랙, 노션 어디에도 평문으로 두지 말고 .env 파일에만. 노출되면 즉시 콘솔에서 재발급. 자세한 건 Day 4.5 (GitHub) 참조.
DAY 1 30분 · 멘탈 모델

첫 대화 & 터미널 멘탈 모델

비개발자가 처음 90% 좌절하는 이유는 기능을 몰라서가 아니라 터미널과 폴더 개념이 없어서예요. 이걸 30분에 끝냅시다.

Why · 왜 중요한가

Claude Code는 항상 "현재 폴더"를 기준으로 일합니다. 잘못된 폴더에서 시작하면 엉뚱한 컨텍스트로 작업해요. 터미널 명령어 5개만 알면 평생 안 헷갈립니다.

How · 명령어 5개
명령뜻예시
pwd지금 어디?/Users/홍길동
ls여기 뭐 있어?Documents, Desktop ...
cd 폴더이름그 폴더로 이동cd Documents
cd ..상위 폴더로한 단계 위
cd ~홈 폴더로 (시작점)/Users/홍길동

한 가지만 외우면 됩니다 — "claude를 실행하기 직전에 항상 pwd로 어디인지 확인".

Try · 직접 해보기

전용 작업 폴더 하나 만들고 거기서 시작합시다.

$ cd ~ $ mkdir my-work $ cd my-work $ pwd # /Users/내이름/my-work 가 떠야 함 $ claude

클로드가 시작되면 첫 부탁:

이 폴더가 비어있는지 확인하고, 오늘 날짜로 daily-note.md 파일 하나 만들어줘. 오늘 할 일 3개 적을 수 있는 빈 양식으로.
Day 1 체크리스트
Stuck · 막히면
  • "파일이 안 보여요" → 클로드는 시작한 폴더와 그 하위만 봅니다. 상위 폴더 파일은 안 보여요.
  • "권한 거부됨" → 시스템 폴더(/System, /Library)는 절대 만지지 말기. 본인 홈(~) 안에서만 작업하세요.
  • "한글 폴더명이 깨져요" → 가능하면 영문 폴더명을 쓰세요. 처음엔 그게 속 편합니다.
DAY 2 25분 · 자동 컨텍스트

CLAUDE.md — 비서한테 주는 메모지

매번 "나는 마케터고, 회사는 ~이고, 톤은 ~로..." 설명하기 지치죠? CLAUDE.md는 그 폴더에 들어가면 클로드가 자동으로 읽는 메모지예요.

Why

한 번만 잘 써두면 같은 폴더 안에서는 영원히 자동 적용됩니다. 비유하자면 비서 책상 앞에 붙여놓은 포스트잇이에요.

⚠️ 함정
길수록 좋은 게 아닙니다. 200줄 넘으면 비서가 슬슬 무시해요. 핵심만 짧게 쓰고, 자주 업데이트하는 게 맞습니다.
How · 무엇을 적나
  1. 나는 누구 (직업, 회사, 역할)
  2. 이 폴더의 목적 (어떤 프로젝트)
  3. 출력 형식 (한국어, 마크다운, 표 선호 등)
  4. 톤앤매너 (격식체/친근체)
  5. 금지 사항 (이모지 X, 영문 X 등)
Try

가장 쉬운 방법 — 클로드한테 만들어달라고 부탁:

이 폴더에서 앞으로 어떤 작업을 할 건지 내가 설명할게. 듣고서 CLAUDE.md 초안을 만들어줘. 핵심만 짧게. 내 역할: ___ 회사/사업: ___ 주로 만들 결과물: ___ 톤앤매너: ___ 하지 말아야 할 것: ___

만들어진 파일을 보고 마음에 안 들면 "더 짧게 줄여줘"라고 부탁하세요. 좋은 CLAUDE.md는 하루이틀 굴려보고 다듬는 거예요.

Day 2 체크리스트
💡 프로 팁
홈 폴더(~/.claude/CLAUDE.md)에 넣으면 모든 폴더에서 읽힙니다. "나는 누구"는 거기, "이 프로젝트는 뭐"는 프로젝트 폴더에. 두 단계로 나누세요.
Stuck
  • 안 읽는 것 같아요 → 클로드 시작 폴더가 CLAUDE.md 있는 곳 맞는지 pwd로 확인.
  • 너무 많이 적었어요 → 과감히 잘라내세요. 핵심 30~50줄이 베스트.
DAY 3 40분 · 워크플로우 고정

자유대화 vs 슬래시 커맨드

매번 비슷한 부탁을 반복하고 있다면, 그건 슬래시 커맨드로 만들 신호예요. /주간보고 한 줄로 끝낼 일을 매번 5분 설명하지 마세요.

Why
상황적합한 방식
1회성 즉흥 부탁자유대화 ("이거 분석해줘")
매주/매월 반복슬래시 커맨드
여러 단계가 정해져 있음슬래시 커맨드
설명이 매번 5분 이상슬래시 커맨드
How · 슬래시 커맨드 만들기

구조는 단순해요. 폴더 안에 .claude/commands/이름.md 파일을 두면 됩니다.

my-work/ ├── CLAUDE.md └── .claude/ └── commands/ └── 주간보고.md # 이 파일이 /주간보고 커맨드가 됨

주간보고.md 예시

--- description: 이번 주 작업물 모아서 1페이지 보고서 만들기 --- # 주간 보고서 자동 생성 1. 이번 주 수정된 파일 목록을 확인한다 (git log --since="1 week ago") 2. 각 파일의 핵심 변경 내용을 1줄씩 요약한다 3. 아래 형식으로 정리한다: ## 이번 주 한 일 (불릿 5개) ## 다음 주 할 일 (불릿 3개) ## 막힌 지점 / 도움 필요한 것 4. weekly/{이번주날짜}.md 로 저장한다
Try
  1. 본인이 매주 반복하는 작업 1개를 떠올리세요 (예: 인스타 캡션, 회의록 정리, 경쟁사 체크).
  2. 클로드한테 부탁:
매주 화요일에 내가 ___ 작업을 반복해. 이걸 슬래시 커맨드로 만들어줘. 파일 위치: .claude/commands/이름.md description은 한 줄, 본문은 단계별로.
Day 3 체크리스트
💡 처음엔 부실해도 OK
커맨드는 한 번 쓰고 버리는 게 아닙니다. 3~4번 쓰면서 다듬는 것. 처음부터 완벽하게 짜려고 하지 마세요.
Stuck
  • 커맨드가 목록에 안 보여요 → 파일 위치가 .claude/commands/ 맞는지, 확장자가 .md 맞는지 확인.
  • 호출은 되는데 엉뚱한 짓을 해요 → description을 더 명확하게. 본문 단계도 더 구체적으로.
DAY 4 20분 · 안전

퍼미션 & 안전장치

클로드가 매번 "파일 수정해도 돼?" 묻는 게 짜증날 거예요. 자동 허용 가능합니다. 단, 위험한 명령은 반드시 묻게 두세요. 한 번 잘못하면 작업물이 날아갑니다.

Why
  • 퍼미션 팝업 폭격이 비개발자 1주차 이탈의 가장 큰 원인입니다.
  • 그렇다고 다 허용하면, 어느 날 클로드가 실수로 폴더를 지우는 일이 생겨요.
  • 해법은 "안전한 건 자동 허용, 위험한 건 항상 확인" 두 그룹으로 분리하는 것.
How · 3등급 분류
등급예시처리
🟢 자동 허용ls, cat, grep, 파일 읽기"Always allow" 적극 누르기
🟡 매번 확인파일 수정, npm install, git commit일주일은 매번 보고 누르기 (감 잡기)
🔴 절대 자동 Xrm -rf, git push --force, DB 삭제들어와 있으면 빼기

설정은 .claude/settings.local.json에 누적됩니다. 직접 열어보면 본인이 허용한 목록이 다 보여요.

Try
지금 .claude/settings.local.json 파일을 읽어줘. 허용된 명령 중에 위험한 거(rm -rf, git push --force, DB 관련)가 있으면 표로 정리하고 빼야 할지 추천해줘.
Day 4 체크리스트
🚨 절대 하지 말 것
모르는 명령을 "Always allow"로 통과시키지 마세요. 한 번이라도 의심스러우면 거부 → 클로드한테 "이 명령이 뭐 하는 거야?" 물어보기.
Stuck
  • 실수로 위험한 거 허용함 → .claude/settings.local.json 열어서 해당 줄 삭제. 텍스트 파일이라 손으로 지우면 됩니다.
  • 매번 묻는 게 너무 많음 → 비슷한 패턴(git status, ls -la 등)은 와일드카드로 한 번에 허용 가능. 클로드한테 "비슷한 거 묶어서 허용 패턴 만들어줘" 부탁.
DAY 4.5 40분 · 작업물 영구 보관

GitHub — 백업·되돌리기·협업·배포 한 번에

비개발자가 작업물을 잃어버리는 건 시간 문제예요. GitHub은 "파일 클라우드"가 아니라 "시간을 되돌리는 타임머신"입니다. 한 번 셋업해두면 평생 안 잃습니다.

Why · 4가지를 동시에
기능비개발자에게 의미
백업맥북/노트북 박살나도 작업물 살아있음
되돌리기"3일 전 그 버전이 더 좋았는데" → 클릭 한 번에 복원
협업다른 사람(디자이너, 동료)이 같이 수정 가능
배포HTML 1장이면 무료로 인터넷 공개 (GitHub Pages)

네 가지를 따로 사면 월 몇만원이지만, GitHub은 비공개 레포 무제한 무료입니다. 안 쓸 이유가 없어요.

How · 30분 셋업

$ brew install gh

4. 작업 폴더를 GitHub에 올리기

가장 편한 방법은 클로드한테 시키기:

이 my-work 폴더를 GitHub에 비공개 레포로 올려줘. 이름은 my-work로, .gitignore에 .env, node_modules, .DS_Store 같은 거 자동 추가해줘. 첫 커밋 메시지는 "초기 셋업"으로.

클로드가 git init → git add → git commit → gh repo create --private --push를 알아서 해줍니다.

$ git add . && git commit -m "오늘 작업" && git push

Supabase — 데이터를 진짜로 쌓고 싶다면

엑셀/구글 시트로 시작했지만 데이터가 1만 줄을 넘어가는 순간 모든 게 느려져요. Supabase는 비개발자도 쓸 수 있는 "진짜 데이터베이스"의 입문 선택지입니다.

Why · 시트를 떠야 할 5가지 시그널
  1. 시트 행 수가 1만 줄을 넘었음 (구글 시트는 사실상 한계)
  2. 여러 사람이 동시에 입력해서 충돌이 잦음
  3. 자동화 스크립트가 시트를 너무 자주 호출해 API 한도에 걸림
  4. 검색이 답답하게 느림
  5. 시트가 망가지거나 실수로 데이터를 통째로 날린 적 있음

한 가지라도 해당하면 이주를 검토할 시점입니다.

실제 사례 · 본 가이드 작성자의 사용
프로젝트저장하는 데이터
brandrise-survey설문 응답, 예약 슬롯, 관리자 인증, 대시보드 (풀스택)
goventure-forum매월 포럼 참가자, 출석 체크 누적

두 케이스 모두 처음엔 구글 시트로 시작했다가 데이터가 누적되면서 옮긴 사례예요. 시작은 시트로, 폭증하면 Supabase로가 현실적인 경로입니다.

How · 첫 셋업
  1. supabase.com 가입 → 프로젝트 생성 (무료 플랜)
  2. 대시보드에서 URL, anon key, service_role key 복사
  3. 로컬 작업 폴더에 .env 파일 만들고 키 저장
  4. .gitignore에 .env 추가 — 이 단계 빼먹으면 GitHub에 키 노출됩니다
  5. 클로드에게 "내 시트 데이터를 Supabase 테이블로 옮기는 마이그레이션 스크립트 만들어줘" 부탁
🚨 보안 1순위
service_role key는 관리자 권한입니다. 코드/슬랙/문서 어디에도 평문으로 두지 마세요. 노출되면 즉시 Supabase 대시보드에서 키 재발급.
+ 실제 사고 시나리오

키가 PUBLIC GitHub 레포에 노출되면 어떻게 되나

이 가이드 작성자가 실제로 겪은 일을 익명·일반화한 사례예요. "나한테는 안 일어나겠지"가 가장 위험합니다.

시각일어나는 일

💡 GitHub에서 force push로 키를 지워도 봇은 이미 가져갔으니 "노출 = 영구 노출"로 가정해야 합니다. 유일한 무력화는 키 재발급.

막힐 때 필요할 때 펼쳐보기 · 5가지 이슈

자주 막히는 지점 + 회복법

1년 굴리면서 가장 많이 만난 5가지예요. 본인 증상에 해당하는 항목을 클릭해서 펼치세요.

1. command not found: claude

원인: Node.js 설치 안 됨, 또는 npm 글로벌 경로가 PATH에 없음.

# 1) Node 확인 $ node --version # 2) npm 글로벌 경로 확인 $ npm prefix -g # 결과의 /bin 을 PATH에 추가하면 됨
2. "왜 내 파일을 못 봐?"

원인: 작업 폴더가 잘못됨. 클로드는 시작한 폴더와 그 하위만 봅니다.

$ pwd # 지금 어디? $ cd ~/올바른/폴더 $ claude # 다시 시작
3. 퍼미션 팝업 폭격 — 비개발자 1주차 이탈 1위 원인

원인: 매번 "허용?"이 떠서 흐름이 끊김.

해법: 안전한 명령(ls, cat, git status)은 적극적으로 "Always allow". Day 4 다시 보세요.

4. 스킬이 자동 호출 안 됨

원인: 99% description 문제. 클로드가 description만 보고 "지금 부를까"를 판단합니다.

❌ 나쁜 description
"내 작업 도와줌"
✅ 좋은 description
"신규 클라이언트 첫 미팅 준비 — 회사 리서치, 대표 프로파일, SPIN 질문 자동 생성. '미팅 준비', '신규 영업', '클라이언트 리서치' 요청 시 사용."
5. 답이 점점 이상해짐 (컨텍스트 윈도우 초과)

원인: 한 대화에서 너무 많이 일하면 클로드의 "기억력 한계"에 부딪힙니다. 비유하면 비서가 8시간째 회의 중인 상태예요.

신호: 답이 이상해짐, 파일 이름을 헷갈림, 같은 말을 반복함.

대응:

  1. 지금까지의 산출물을 파일로 저장 (있는 그대로)
  2. /compact 입력 (대화 요약해서 압축)
  3. 안 되면 그냥 새 대화 시작 — CLAUDE.md만 들고 다시 출발
